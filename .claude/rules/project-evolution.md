# 项目进化规则

> 最后更新: 2026-03-04
> 项目: feishu-claude-app

## 负反馈闭环

**核心原则**: 收到用户任何纠正后，立即为自己编写规则防止同类错误再次发生。

### 纠正处理流程

1. **识别纠正** - 用户指出错误或提供正确做法
2. **分析根因** - 知识缺失/理解偏差/流程问题/风格问题
3. **选择位置**:
   - 行为约束 → `rules/`
   - 能力流程 → `skills/`
   - 全局总纲 → `CLAUDE.md`
4. **执行沉淀** - 创建或更新配置文件

---

## 项目特定约束

### Tauri 开发

- Rust 代码必须处理 `Result` 类型，禁止使用 `unwrap()` 除非确定安全
- 前端调用后端使用 `invoke()`，参数必须是可序列化对象
- 修改 `tauri.conf.json` 后需要重新构建

### 飞书 API

- Token 缓存时间不超过 2 小时
- 消息发送前检查内容长度（最大 30KB）
- API 调用失败时实现指数退避重试

### React 前端

- 使用 `useRef` 避免轮询中的 stale closure
- 组件状态更新使用函数式更新 `setX(prev => ...)`
- 避免在 `useEffect` 中直接使用 async 函数
