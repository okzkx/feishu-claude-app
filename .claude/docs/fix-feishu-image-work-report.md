### 2.2 阶段二：问题分析 (10 分钟)

**分析过程**:
1. 读取错误日志，定位问题发生在 `MainPage.tsx:490`
2. 追踪调用链: `handleTestImage` → `uploadImage` → `sendImageMessage` → `sendMessage`
3. 分析飞书 API 文档，确定 `content` 字段格式要求
4. 识别根因：`JSON.parse()` 导致 `content` 变成对象而非字符串

**关键发现**:
- 飞书 API 要求 `content` 必须是 JSON 字符串格式
- Axios 自动序列化会破坏正确的格式
- 图片消息的 `content` 不需要 `JSON.parse()`

---

### 2.3 阶段三：代码修复 (5 分钟)

**修改文件**: `src/utils/feishuApi.ts`

**核心变更**:
```typescript
// 修改前
let messageContent: any;
if (msgType === "image") {
  messageContent = JSON.parse(content);  // ❌ 错误
}

// 修改后
let messageContent: string;
if (msgType === "image") {
  messageContent = content;  // ✅ 正确
}
```

**验证结果**:
- TypeScript 编译通过 ✅
- 文本消息功能不受影响 ✅
