
### 3.2 E2E 测试策略

1. **关键路径测试**: 覆盖用户主要使用场景
2. **边界条件测试**: 测试输入边界和异常情况
3. **状态转换测试**: 测试应用状态变化
4. **UI 交互测试**: 测试按钮、表单等交互

### 3.3 截图测试策略

1. **基准截图**: 在首次发布时建立
2. **对比阈值**: 允许 1% 像素差异
3. **忽略区域**: 动态内容区域（时间戳等）
